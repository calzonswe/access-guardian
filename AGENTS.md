# AGENTS.md — Access Guardian (RBAC)

## Project overview

RBAC access control system with a React frontend and Express/PostgreSQL backend. Generated by [Lovable](https://lovable.dev) — commits push back to Lovable automatically.

## Commands

| Task | Command |
|------|---------|
| Frontend dev | `npm run dev` (port 8080) |
| Frontend build | `npm run build` |
| Frontend lint | `npm run lint` |
| Frontend unit tests | `npm run test` (vitest, single run) / `npm run test:watch` |
| Backend dev | `cd backend && npm run dev` (port 3000, `node --watch`) |
| Full stack (Docker) | `docker compose up` (needs `.env` — copy from `.env.example`) |

## Architecture

```
frontend (React/Vite, port 8080)  →  backend (Express, port 3000)  →  PostgreSQL (rbac_access)
```

- **Frontend**: `src/` — React 18, TypeScript, shadcn-ui, Tailwind, React Query, React Router
- **Backend**: `backend/src/` — Express.js (plain JS, not TS), `pg` driver, JWT auth
- **Database**: `db/init.sql` — full schema + seed, loaded by Docker on first run
- **Docker**: `docker-compose.yml` runs 3 services: `db` → `backend` → `frontend` (nginx)
- **Backend middleware**: `auth.js` (JWT), `rbac.js` (role-based auth helpers), `rateLimit.js` (login throttling)
- **Backend routes**: `settings.js`, `attachments.js` added to index.js alongside existing 12 route files

## Key conventions

- **Path alias**: `@/*` maps to `./src/*` (configured in `vite.config.ts`, `tsconfig.json`, `vitest.config.ts`)
- **TypeScript is relaxed**: `noImplicitAny: false`, `strictNullChecks: false`, `noUnusedLocals: false` — do not tighten without asking
- **ESLint**: `@typescript-eslint/no-unused-vars` is turned off; `react-refresh/only-export-components` is warn-level
- **Backend is plain JS** — no TypeScript, no build step, runs directly with `node`
- **API proxy**: nginx proxies `/api` to the backend in production; frontend uses relative `/api/*` paths

## RBAC roles

`administrator` | `facility_owner` | `facility_admin` | `line_manager` | `employee` | `contractor`

Routes are guarded by `RoleGuard` in `App.tsx` — check there for which pages each role can access.

## Testing

- **Unit**: vitest with jsdom, tests live in `src/**/*.{test,spec}.{ts,tsx}`, setup in `src/test/setup.ts`
- **E2E**: Playwright via `lovable-agent-playwright-config`, config in `playwright.config.ts`, fixture re-exported from `playwright-fixture.ts`

## Env vars (`.env`)

Required for Docker: `DB_PASSWORD`, `JWT_SECRET`. Optional: `APP_PORT` (default 8080), `VITE_API_URL`.

## Notes

- Lovable platform auto-syncs commits — changes made on lovable.dev are pushed to this repo
- `lovable-tagger` dev plugin is active in development mode only
- No `package-lock.json` in backend/ — it uses its own `npm install --production` in Docker
- Backend has RBAC enforcement: `requireRole()` and facility-level `requireFacilityAccess()` middleware protect all routes
- Application workflow: `draft` → `pending_manager` (line_manager approves) → `pending_facility` (facility_owner/admin approves) → `approved`
- Exception requests go to `pending_exception` → administrator approves
- Attachments are uploaded as base64 via `POST /api/attachments` (JSON body, max 10MB per file)
- System settings are stored in `system_settings` table, managed via `GET/PUT /api/settings`
